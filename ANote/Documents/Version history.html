 BUGGFIX: In version 4.1 the alarm-button was accidently not visible when setting an alarm. Fixed now.
 BUGGFIX: Some of the toolbar-buttons wasn't painted correctly. Fixed.

4.1
 2004-10-15

 

 Moved around a bit in the options-dialog.
The user can select what will happend when pressing only one time on the tray icon with the left mouse-button. Either a new note will appear  or all notes will either be hidden or shown.
If the screen resolution changes the notes will move accordingly.
The FAQ received a grammar and spelling check. Thanks Kristin.
Added Strikeout as a toolbar-button
Clicking on the alarm-button will show the alarm-dialog (if not the alarm is shouting, then you will see the snooze/turn off dialog).
The readable text (not RTF) is sent to the server. In case we want to read the note's either in a cell-phone or on the web. But that is to come, but I have merely added this possibillity now.
Added language support. Right now only Swedish and English is supported. The user can however easily change or add new languages.
The silded notes will have the same amount of translucensy as if they were minimized.
The toolbar-buttons can be position where you like it to.
BUGGFIX: Sometime when creating a new note, the new note received wrong note-id, causing the new note to disappear when exit the program.
BUGGFIX: When locking the computer with Ctrl+Alt+Del the slided notes rezised strangly. Fixed

4.0.1
 2004-10-02 BUGGFIX: When sliding the notes to the side, the height of the slide was the same height as the note is in normal mode, but instead it should be the height of the title's text plus 30 pixels.
BUGGFIX: This shift-delete feature only worked sometimes. Now fixed.
 
4.0
 2004-10-01
 When the notes are deleted they will be saved in a new file. By selecting 'Recycle bin' from the tray-menu, you will be able to view the deleted notes and either permanently delete them or restore them.
In the options menu one can set how many deleted notes that will be saved and how they will be sorted
When deleting notes the note will NOT be saved if the use presses the shift-key at the same time.
It can be set in the options - database that the program won't start downloading the notes immediately. This feature can be nice to use when it takes some time before you have an internet connection.
The note popup-menu works on the title aswell
You can check if any newer version exists from the tray-menu
Changed the layout of the close, minimize and bullet buttons
The user can select which toolbar button that should be visible
The notes can be slided to the right or left (via toolbar-button)
BUGGFIX: The value in the registry for move the mouse when minimizing the notes has been fixed.

3.3 2004-09-28 Added documentation to create and use a note-server of your own. (see Server Documents)

An errorlog has been added. So when a problem occur a log will be created for that in the application data folder.

BUGGFIX: The program will read the notes that are saved on disk. This has been a huge bug and is now fixed

3.0

2004-04-06

By pressing the bullet-button, the text in the note can be bulleted.
The data will be saved in the user's application path (i.e c:\documents and settings\user\application data\a note)
The user can set which server the notes will be uploaded to. In that way an own server can be used
It can be set wether the cursor should move to the minimize-button when the notes are minimized.
The user can read the FAQ from the A Note menu.
BUGGFIX: When A Note started any minimized note would make the cursor to jump around. This is fixed now.
 

2.1

2004-02-17

Hot Keys can be set. When a hot key is set the user can create a new note by pressing i.e. CTRL + ALT + N. Naturally the user can set what keys that should be hot keys.
 

2.0

2003-11-18

It can be set what will happend when doubleclicking on the titlebar. Either the user can set the titel or the note will be restored/minimized
All notes that are minimized can be automatically restored in one way, via a menuitem.
The notes on the server can now be deleted by the user (in the options menu).
Added two transparancy-modes. One when the notes are in normal mode and one when the notes are minimized but still visible. This feature can be set in the options-menu.
